- Git 管理下で作業を行い、主要な実験ごとにコミットを記録してください。

## 2. 仮説駆動アプローチの手順
1. 新しい実験を行う前に、**仮説を明文化**してください。  
 - 仮説の形式：  
   > 「特徴量 X を追加すると、ターゲット Y との相関が高まり、交差検証スコアが改善するはず」  
 - 仮説には以下を含めてください：  
   - 変更対象（前処理／特徴量／モデル構造／ハイパーパラメータ）  
   - 期待効果・理由  
   - 評価指標（例：AUC、F1、RMSE など）  
2. 仮説に基づき **変更を一つに絞った実験** を行ってください。  
 - 一度に複数の変更を加えると、「効果の原因」が不明確になります。  
3. 実験後、結果を **定量的に記録**してください。  
 - 例：交差検証 AUC 0.812 → 0.829、ホールドアウトスコア 0.805 → 0.821  
 - 改善が見られない場合でも「なぜ効果が出なかったか」理由を併記してください。  
4. 成果が確認できればメインブランチにマージし、次の仮説へ移行してください。  
 - 効果が小さい／コストが高い場合は実験を中断または仮説を再設定してください。

## 3. コーディング・スタイル
- 変数・関数命名は `snake_case`（Python の慣例）を基本とします。  
- クラス命名は `PascalCase` を使用。  
- ドキュメント文字列（docstring）を必ず記述してください。特に新規クラス・関数には必要です。  
- .py ファイルの最大行数は 300 行を目安とし、処理はモジュール分割を心がけてください。  
- 不要な外部ライブラリの導入は避け、標準ライブラリ＋既存ライブラリを優先してください。

## 4. 実験・評価
- 交差検証／ホールドアウト分割を必ず実施してください。  
- ハイパーパラメータ調整時、必ず **独立した検証セット**を保持し、過学習を防止してください。  
- 評価指標はコンペメントのもの（提出スコア）を明確に記録し、内部指標（例えば交差検証スコア）との乖離をモニタリングしてください。  
- 実験毎に **実験ID／仮説番号を付与**し、実験の比較・検索を容易にしてください。  
- 特徴量の生成・モデル構築・評価にいたる全ログを保存し、再現可能性を担保してください。
